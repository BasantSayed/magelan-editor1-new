Bugs

magelan.core.entity.PolyLine

private void newShape()

The GeneralPath.moveTo(float, float) has no equivalent that takes double parameters. This means that when new shapes are being constructed their model coordinates will be effectively truncated to floats!

magelan.core.editor.DefaultDrawingEditor

private boolean isPointed(EntityModifier, double, double)

The method takes model space coordinates to check if a handle is pointed or not. This way the size of the checked area will be dependent on drawing's scale. Should be changed to comparing transformed screen coordinates; also in drawHandles.

Tasks

magelan.core.editor.DrawingEditor

public int getMode()

Probably there must be a mode change listener mechanism. Currently mode changes are processed and listened to at the level of the editor project in a somewhat strange manner. This issue is related to a possible extension of the IStatusSource interface to process general status change events, not only in the light of displaying them to a StatusBar. [?]

magelan.core.editor.DefaultSnap

public void propertyChange(PropertyChangeEvent)

In order to perform updates of the internal list of modifiers more efficiently, listen to the specific types of property change events like add or remove and update only the affected entities, not all. Currently a full update is performed every time a property change occurs. The list of modifiers should probably become a LinkedList.

magelan.core.editor.DefaultDrawingEditor

public int getScrollableUnitIncrement(Rectangle, int, int)

Make the unit increment a percentage of drawing's size. The percentage may be stored as a property using putValue and getValue.

public int getScrollableBlockIncrement(Rectangle, int, int)

Make the block increment a percentage of drawing's size. The percentage may be stored as a property using putValue and getValue.
