Deployment of Extensions
Menu:   Extensions | Manage

Deployment of a single extension

Select the Manifests tab and add the manifest file name to the list of known manifests. If the file cannot be located an error icon will appear next to its name.

Deployment of multiple extensions

Select the Paths tab and add the directory the manifest file is located in. Again if the directory cannot be found an error icon will appear next to it. To add all manifests located in the listed directories to the list of known manifests, choose the Scan button. This option is useful for deployment of multiple extensions.

Automatic loading of extensions

If the Auto find and load option from the Preferences | Workspace dialog is selected the editor will ignore the manifests listed in the Manifests tab of the extension manager dialog. It will instead scan the directories listed in the Paths tab (much as the Scan button will) for manifest files and load their corresponding extensions. The drawback is that the user cannot influence the order the extensions will appear in the editor the next time it is started. This option is turned off by default.

Changing the order of loading of extensions

The order of manifest files (which is also the order they will be read and the extensions loaded and placed in the toolbars the next time the editor is started) can be changed by dragging them with the right mouse button. To edit a manifest, choose the Edit button
