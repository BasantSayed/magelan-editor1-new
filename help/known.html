Reporting Bugs

Bugs, bugs, bugs

Help us improve this software! It certainly contains bugs and you will certainly stumble on some of them. Please report them. Even when you think that the bug is known reporting it will help us assess how often the issue occurs and properly prioritise it.

Caught errors

We have done our best to catch errors in places they are likely to occur. When an error occurs a dialog titled Error Notification or Fatal Error appears. It contains a message regarding the error, details about it and a Web address of our bug submission form. If the error is unexpected and you know it was not a result of your actions please copy the error details to the bug submission form. You may also submit the contents of the magelan.log and exception.log files.

Privacy

Error details and the exception.log file contain error traces and information about the versions of Magelan, JRE, operating system and hardware you use. This information will help us find out why the error occured. No confidential details about the person using the machine are intentionally collected. Please, when submitting a bug report, delete from the error trace any information you do not feel like revealing!

Uncaught errors

It is not possible to catch all errors and sometimes the editor behaves strangely but no error message occurs. In this case we will be grateful if you describe the situation in the bug submission form, along with the details you find relevant and the contents of the exception.log and magelan.log files.
